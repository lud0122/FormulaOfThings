# 轮廓提取改进说明

## 问题背景

`ideal/ideal_1.PNG` 是一只展翅高飞的鸟的轮廓图，但轮廓线有部分缺失（断裂）。原有的轮廓提取算法只能找到5个噪点，无法识别完整的鸟轮廓。

## 实施的改进

### 1. 形态学修复模块 (`src/image-processor/morphology.js`)

**功能：**
- **膨胀操作**：扩展黑色像素（轮廓线），连接断裂的部分
- **腐蚀操作**：收缩黑色像素，去除细小噪点
- **闭运算**：先膨胀后腐蚀，连接断裂轮廓的同时保持形状

**参数：**
- 结构元素：5x5 十字形（比3x3更强的连接能力）
- 迭代次数：根据背景像素比例自适应调整
  - 白色背景 > 90%：5次迭代（强力修复）
  - 白色背景 70-90%：4次迭代
  - 白色背景 50-70%：3次迭代
  - 白色背景 < 50%：2次迭代

**效果：**
- 修复前：黑色像素 111,831（占5.1%）
- 修复后：黑色像素 180,872（占8.2%）
- 增加：69,041个像素，成功连接断裂的轮廓线

### 2. 全像素提取函数 (`extractAllBlackPixels`)

**用途：**
当传统轮廓追踪失败或找到的点数过少时，直接提取所有黑色像素作为轮廓点。

**参数：**
- `binaryImage`: 二值图像
- `sampleStep`: 采样步长（默认每10个像素采样一次）

**返回：**
- 轮廓点数组 `Array<{x, y}>`

### 3. 智能回退逻辑

```javascript
// 尝试传统轮廓追踪
try {
  contour = traceContour(repairedData)
  console.log(`[轮廓提取] 传统追踪完成, 点数: ${contour.length}`)

  // 如果点数太少，改用全像素提取
  if (contour.length < 50) {
    console.log(`[轮廓提取] 点数过少，改用全像素提取模式`)
    contour = extractAllBlackPixels(repairedData, 10)
  }
} catch (error) {
  // 失败时使用全像素提取
  console.log(`[轮廓提取] 传统追踪失败，改用全像素提取模式`)
  contour = extractAllBlackPixels(repairedData, 10)
}
```

## 处理流程

```
用户上传图像
  ↓
图像类型检测（contour类型）
  ↓
Otsu自动阈值计算（阈值=85）
  ↓
二值化 + 自动反转
  ↓
形态学修复（5x5结构元素，5次迭代）
  ↓
尝试传统轮廓追踪
  ├─ 成功且点数≥50 → 使用追踪结果
  └─ 失败或点数<50 → 全像素提取
  ↓
傅里叶分析
  ↓
轮圆动画渲染
```

## 技术要点

1. **修正形态学方向**：扩展黑色像素（轮廓），而非白色像素（背景）
2. **自适应修复强度**：根据背景像素比例动态调整迭代次数
3. **双保险策略**：传统追踪优先，全像素提取作为fallback
4. **合理的采样步长**：全像素提取时每10个像素采样，平衡精度和性能
