# 公式生成图像（基于参考图）的设计说明

## 背景
用户目标：基于 `ideal/ideal_1.PNG`，实现一个可在网页运行的公式生成系统，要求：
- 先实现高还原（主观视觉一致）
- 再支持参数可调与生成扩展
- 公式可解释
- 不允许手工锚点，必须纯公式自动推导
- 当前实现路线采用 Canvas 2D（兼容性与实现效率优先）

## 需求确认结论
1. 渲染内核：Canvas 2D
2. 目标模式：两阶段（高还原 + 可调生成）
3. 约束：纯公式自动推导，不使用手工控制点
4. 验收：以主观视觉一致为主
5. 部署：按仓库结构自动判断入口，走默认 GitHub Pages 流程

## 总体架构
系统分为四层：

1) 参考图解析层
- 输入：`ideal/ideal_1.PNG`
- 输出：标准化视觉特征（亮度分布、径向统计、边缘密度、方向纹理）
- 作用：将图像目标转化为可优化对象

2) 公式模型层
- 以可解释函数族构成生成模型：
  - 径向基项（控制整体明暗框架）
  - 极坐标波项（控制环纹/辐射节奏）
  - 域噪声调制项（控制局部复杂度）
  - 色调映射项（控制视觉风格输出）
- 统一表达：`I(x, y, t) = F(params)`

3) 自动拟合层
- 参数搜索：随机重启 + 局部微调
- 目标：最小化“主观一致代理指标”
- 输出：最佳参数集 `bestParams`

4) 渲染与交互层
- Canvas2D 实时渲染
- 参数面板支持分组调节（几何/材质/颜色/动画）
- 支持导出图像与参数

## 模块与文件规划
- `index.html`：页面骨架
- `styles/main.css`：基础样式
- `src/app/main.js`：应用入口与生命周期
- `src/formula/model.js`：公式定义与参数到图像映射
- `src/formula/params.js`：参数 schema（默认值/范围/步进）
- `src/fit/extractFeatures.js`：参考图特征提取
- `src/fit/score.js`：特征差异评分
- `src/fit/optimizer.js`：自动拟合器
- `src/render/canvasRenderer.js`：Canvas2D 渲染器
- `src/ui/controlPanel.js`：参数 UI 绑定
- `src/io/referenceLoader.js`：参考图读取与标准化
- `src/io/exporter.js`：导出 PNG 与参数 JSON
- `ideal/ideal_1.PNG`：参考图输入（已存在）

## 数据流
1. 加载参考图并提取目标特征 `targetFeatures`
2. 自动优化器生成候选参数
3. 用候选参数渲染预览并提取候选特征
4. 评分并保留最优参数
5. 用最优参数全分辨率渲染
6. 用户调参、导出、复现

## 错误处理策略
- 参考图加载失败：给出路径/格式提示
- 参数越界：按 schema 钳制
- 优化超时：返回当前最优并提示可继续
- 渲染异常：保留上一次成功结果并提示

## 验收标准
主验收：主观视觉一致
- 初始结果在构图、明暗节奏、主纹理上接近参考图
- 参数变更具备可解释视觉响应

辅助验收：可运行与可复现
- 页面可打开并完成拟合与渲染
- 导入导出参数后可复现结果

## 说明
本设计先聚焦可解释与可调的稳定实现，不引入 WebGL/Shader 复杂性；后续可在不改变参数语义的前提下迁移渲染后端。
